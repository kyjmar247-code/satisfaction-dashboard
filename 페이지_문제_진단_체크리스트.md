# GitHub Pages 배포 문제 진단 체크리스트

## 1단계: GitHub Pages 설정 확인 ✅
- [ ] GitHub 저장소 페이지로 이동: https://github.com/kyjmar247-code/satisfaction-dashboard
- [ ] **Settings** 탭 클릭
- [ ] 왼쪽 메뉴에서 **Pages** 클릭
- [ ] **Source**에서 **GitHub Actions** 선택되어 있는지 확인
- [ ] 저장 (이미 설정되어 있다면 넘어가기)

## 2단계: GitHub Actions 확인 ✅
- [ ] GitHub 저장소에서 **Actions** 탭 클릭
- [ ] "Deploy to GitHub Pages" 워크플로우가 보이는지 확인
- [ ] 워크플로우를 클릭해서 빌드 상태 확인:
  - ✅ **초록색 체크**: 배포 성공
  - ❌ **빨간색 X**: 배포 실패 → 오류 메시지 확인 필요
  - ⏳ **노란색 원**: 진행 중 → 완료될 때까지 대기

## 3단계: 사이트 URL 확인 ✅
올바른 URL로 접속했는지 확인:
- ✅ **올바른 URL**: `https://kyjmar247-code.github.io/satisfaction-dashboard/`
- ❌ **잘못된 URL**: `https://kyjmar247-code.github.io/`

## 4단계: 브라우저 캐시 문제 확인 ✅
- [ ] 브라우저에서 **Ctrl + Shift + R** (강력 새로고침)
- [ ] 또는 브라우저 캐시 삭제 후 재접속

## 5단계: 오류 확인 ✅
- [ ] 브라우저에서 **F12** 키 누르기 (개발자 도구)
- [ ] **Console** 탭에서 빨간색 오류 메시지 확인
- [ ] **Network** 탭에서 CSV 파일 로드 실패 여부 확인

## 일반적인 문제와 해결 방법

### 문제 1: Actions 탭에 워크플로우가 없음
**해결**: 워크플로우 파일이 푸시되었는지 확인
- `.github/workflows/deploy.yml` 파일이 있는지 확인
- 파일이 없다면 다시 푸시

### 문제 2: 빌드 실패 (빨간색 X)
**해결**: 
1. Actions 탭에서 실패한 워크플로우 클릭
2. 빌드 실패 원인 확인:
   - CSV 파일 누락
   - package.json 문제
   - 빌드 에러
3. 오류 메시지를 보고 수정

### 문제 3: 404 에러 (페이지를 찾을 수 없음)
**해결**:
1. URL이 정확한지 확인: `https://kyjmar247-code.github.io/satisfaction-dashboard/`
2. GitHub Pages 설정 확인 (1단계)
3. 배포가 완료되었는지 확인 (2단계)

### 문제 4: 페이지는 열리지만 데이터가 안 보임
**해결**:
1. 브라우저 콘솔(F12)에서 오류 확인
2. CSV 파일 경로 문제일 수 있음
3. Network 탭에서 CSV 파일 로드 여부 확인
