# 연결 문제 해결 가이드

## "사이트에 연결할 수 없다" 오류

### 1단계: 서버가 실행 중인지 확인

1. **터미널/명령 프롬프트 확인**
   - `빠른_시작.bat`를 실행한 창을 확인
   - 다음과 같은 메시지가 보여야 함:
     ```
     VITE v5.x.x  ready in xxx ms
     ➜  Local:   http://localhost:3000/
     ➜  Network: http://192.168.x.x:3000/
     ```

2. **서버가 실행되지 않았다면**
   - 에러 메시지를 확인
   - 일반적인 에러:
     - `EADDRINUSE`: 포트 3000이 이미 사용 중
     - `Cannot find module`: `npm install` 필요
     - CSV 파일 오류: CSV 파일 경로 확인

### 2단계: 포트 충돌 확인

포트 3000이 이미 사용 중일 수 있습니다.

**해결 방법:**

1. **다른 포트 사용**
   - `vite.config.js` 파일 열기
   - `port: 3000`을 `port: 3001`로 변경
   - 저장 후 서버 재시작
   - 브라우저에서 http://localhost:3001 접속

2. **포트 사용 중인 프로그램 종료**
   ```bash
   netstat -ano | findstr :3000
   taskkill /PID [프로세스ID] /F
   ```

### 3단계: CSV 파일 경로 확인

CSV 파일이 올바른 위치에 있는지 확인:

1. **파일 위치 확인**
   ```
   dashboard/
   └── public/
       └── converted/
           ├── 2025 하반기 재원생 만족도조사 데이터 및 문항속성_챗GPT_데이터.csv
           └── 2025 하반기 재원생 만족도조사 데이터 및 문항속성_챗GPT_문항 및 보기.csv
   ```

2. **CSV 파일 복사**
   - `CSV_파일_복사.bat` 실행
   - 또는 수동으로 복사

### 4단계: 브라우저 콘솔 확인

1. **브라우저에서 F12 키 누르기**
   - 개발자 도구 열림

2. **Console 탭 확인**
   - 빨간색 에러 메시지 확인
   - 에러 내용을 기록

3. **Network 탭 확인**
   - CSV 파일 요청이 실패했는지 확인
   - 404 또는 500 에러가 있는지 확인

### 5단계: 수동 접속

자동으로 열리지 않으면:

1. **서버가 실행 중인지 확인**
   - 터미널에서 "Local: http://localhost:3000" 메시지 확인

2. **브라우저에서 직접 접속**
   - 주소창에 입력: `http://localhost:3000`
   - 또는 `http://127.0.0.1:3000`

### 6단계: 방화벽 확인

Windows 방화벽이 차단할 수 있습니다:

1. **Windows 보안 설정 열기**
   - 설정 → 개인 정보 보호 및 보안 → Windows 보안

2. **방화벽 설정 확인**
   - 앱이 차단되지 않았는지 확인

### 7단계: 재설치

문제가 계속되면:

1. **node_modules 폴더 삭제**
   ```bash
   cd dashboard
   rmdir /s /q node_modules
   ```

2. **의존성 재설치**
   ```bash
   npm install
   ```

3. **서버 재시작**
   ```bash
   npm run dev
   ```

## 자주 발생하는 에러

### "Cannot find module 'xxx'"
**해결:** `npm install` 실행

### "Port 3000 is already in use"
**해결:** `vite.config.js`에서 포트 번호 변경 (예: 3001)

### "CSV 파일을 불러올 수 없습니다"
**해결:** 
- `public/converted/` 폴더에 CSV 파일이 있는지 확인
- 파일명이 정확한지 확인 (특수문자, 공백 포함)

### "페이지를 찾을 수 없습니다" (404)
**해결:**
- 서버가 실행 중인지 확인
- 올바른 URL인지 확인 (http://localhost:3000)

### 빈 페이지 또는 로딩만 계속됨
**해결:**
- F12 키로 콘솔 확인
- CSV 파일 경로 확인
- 네트워크 탭에서 파일 로드 실패 확인

## 도움 요청 시 제공할 정보

문제가 해결되지 않으면 다음 정보를 제공해주세요:

1. **에러 메시지** (전체)
2. **터미널 출력** (서버 시작 메시지)
3. **브라우저 콘솔 에러** (F12 → Console)
4. **CSV 파일 위치** 확인
5. **Node.js 버전**: `node --version`
6. **npm 버전**: `npm --version`
